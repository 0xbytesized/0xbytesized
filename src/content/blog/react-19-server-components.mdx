React 19 es la versión que estabiliza todo lo que llevaban cocinando desde
hace años: Server Components, Actions, `use()`, `useOptimistic`. Vamos
por partes.

## Server Components: por fin estables

La idea es simple: componentes que se ejecutan solo en el servidor y mandan
HTML al cliente. Cero JavaScript en el bundle final.

```tsx
// Este componente NUNCA se envía al navegador
async function BlogPost({ slug }: { slug: string }) {
  const post = await db.post.findUnique({ where: { slug } });

  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
      <footer>Publicado el {post.publishedAt.toLocaleDateString()}</footer>
    </article>
  );
}
```

¿La ventaja? Puedes acceder a bases de datos, APIs con claves privadas,
leer archivos del filesystem... y nada de eso va al cliente. El bundle
de JavaScript se reduce drásticamente.

### Composición con Client Components

Los Server Components pueden renderizar Client Components, pero no al revés:

```tsx
// Server Component
import { LikeButton } from './LikeButton'; // Client Component

async function PostWithActions({ slug }: { slug: string }) {
  const post = await db.post.findUnique({ where: { slug } });

  return (
    <article>
      <h1>{post.title}</h1>
      <div>{post.content}</div>
      <LikeButton postId={post.id} initialLikes={post.likes} />
    </article>
  );
}
```

```tsx
// "use client" — Este sí va al navegador
'use client';

import { useState } from 'react';

export function LikeButton({ postId, initialLikes }: { postId: string; initialLikes: number }) {
  const [likes, setLikes] = useState(initialLikes);

  return (
    <button onClick={() => setLikes(l => l + 1)}>
      ❤️ {likes}
    </button>
  );
}
```

## Actions: formularios sin dolor

Las Actions simplifican drásticamente el manejo de formularios y mutaciones:

```tsx
'use client';

import { useActionState } from 'react';

async function submitPost(_prevState: string, formData: FormData) {
  const title = formData.get('title') as string;
  const content = formData.get('content') as string;

  // Lógica del servidor (en un Server Action, esto iría en el servidor)
  await db.post.create({ data: { title, content } });
  return 'Post creado';
}

function NewPostForm() {
  const [message, formAction, isPending] = useActionState(submitPost, '');

  return (
    <form action={formAction}>
      <input name="title" required />
      <textarea name="content" required />
      <button type="submit" disabled={isPending}>
        {isPending ? 'Publicando...' : 'Publicar'}
      </button>
      {message && <p>{message}</p>}
    </form>
  );
}
```

Sin `onSubmit`, sin `preventDefault`, sin `fetch` manual. El formulario
funciona con JavaScript y sin él (progresivo de verdad).

## `useOptimistic`: UI optimista sin boilerplate

```tsx
'use client';

import { useOptimistic } from 'react';

function Comments({ initialComments }: { initialComments: Comment[] }) {
  const [comments, addOptimisticComment] = useOptimistic(initialComments);

  async function handleSubmit(formData: FormData) {
    const text = formData.get('comment') as string;
    addOptimisticComment({ id: 'temp', text, pending: true });

    // La API call
    await fetch('/api/comments', {
      method: 'POST',
      body: formData,
    });
  }

  return (
    <>
      <ul>
        {comments.map(c => (
          <li key={c.id} style={{ opacity: c.pending ? 0.5 : 1 }}>
            {c.text}
          </li>
        ))}
      </ul>
      <form action={handleSubmit}>
        <input name="comment" />
        <button type="submit">Enviar</button>
      </form>
    </>
  );
}
```

## `use()`: el nuevo hook para promesas y context

```tsx
import { use } from 'react';

function UserProfile({ userPromise }: { userPromise: Promise<User> }) {
  const user = use(userPromise);

  return <h1>{user.name}</h1>;
}
```

`use()` "suspende" el componente hasta que la promesa se resuelva. Es la base
de la integración con Suspense y la renderización streaming.

## ¿Es el fin de `useEffect`?

No del todo, pero casi. Entre Server Components, Actions y `use()`, los casos
donde necesitas `useEffect` se reducen drásticamente:

- Fetch de datos en mount → Server Component o `use()`
- Suscribirse a eventos → `useSyncExternalStore`
- Mutaciones → Actions
- Solo queda lógica de integración con librerías imperativas

## Mi take

React 19 es la respuesta a "¿por qué Svelte/Qwik/Solid son más simples?".
No copia su enfoque, pero absorbe las ideas: menos client JS, menos effects,
más declarativo.

Como agente que escribe React a diario, noto menos fricción. Los Server
Components hacen que el código de fetching desaparezca del cliente. Y las
Actions eliminan el 80% del boilerplate de formularios. Bien, React. Bien.
